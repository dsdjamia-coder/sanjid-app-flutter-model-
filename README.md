 നിലവിലുള്ള Repository കോപ്പി ചെയ്തുകൊണ്ട് (അല്ലെങ്കിൽ Fork ചെയ്ത്) **പുതിയൊരു പേരിൽ** മറ്റൊരു പ്രോജക്റ്റ് തുടങ്ങുകയാണെങ്കിൽ, **ഓട്ടോമാറ്റിക് ആയി എല്ലാം വർക്ക് ആകില്ല.**

പ്രധാനമായും **3 സ്ഥലങ്ങളിൽ** നിങ്ങൾ മാറ്റം വരുത്തേണ്ടതുണ്ട്. എങ്കിൽ മാത്രമേ പുതിയ ആപ്പിന്റെ വെബ്സൈറ്റും APK-യും ശരിയായി ലഭിക്കുകയുള്ളൂ.

അത് എന്തൊക്കെയാണെന്ന് താഴെ നൽകുന്നു:

### 1. `build.yml` ഫയലിലെ മാറ്റം (ഏറ്റവും പ്രധാനം)

നിങ്ങൾ പുതിയ Repository-ക്ക് **"new-project"** എന്ന് പേര് നൽകി എന്ന് കരുതുക. പക്ഷെ നിങ്ങളുടെ `build.yml` ഫയലിൽ ഇപ്പോഴും പഴയ ലിങ്ക് ആയിരിക്കും ഉണ്ടാവുക. ഇത് വെബ്സൈറ്റ് വർക്ക് ആകാതിരിക്കാൻ കാരണമാകും.

* **എവിടെ മാറ്റണം?:** `.github/workflows/build.yml`
* **എന്ത് മാറ്റണം?:**
താഴെ കാണുന്ന വരി കണ്ടുപിടിക്കുക:
`run: flutter build web --release --base-href "/sanjid-app-flutter-/"`
ഇതിലെ `"/sanjid-app-flutter-/"` എന്നത് മാറ്റി നിങ്ങളുടെ പുതിയ Repository-യുടെ പേര് നൽകുക.
ഉദാഹരണത്തിന്: `"/new-project/"`

---

### 2. GitHub Settings (പെർമിഷൻ നൽകൽ)

പുതിയ Repository തുടങ്ങുമ്പോൾ പഴയതിന്റെ സെറ്റിംഗ്സ് അതിലേക്ക് കോപ്പി ചെയ്യില്ല. അതിനാൽ "Action" വർക്ക് ആകാൻ പെർമിഷൻ വീണ്ടും നൽകണം.

* **എവിടെ മാറ്റണം?:** GitHub Settings -> Actions -> General -> Workflow permissions.
* **എന്ത് മാറ്റണം?:** **"Read and write permissions"** എന്നത് വീണ്ടും സെലക്ട് ചെയ്ത് Save ചെയ്യുക. (ഇത് ചെയ്തില്ലെങ്കിൽ APK അപ്‌ലോഡ് ആകില്ല).

---

### 3. GitHub Pages (വെബ്സൈറ്റ് ഓൺ ആക്കൽ)

പുതിയ പ്രോജക്റ്റിൽ വെബ്സൈറ്റ് ഓട്ടോമാറ്റിക് ആയി ഓൺ ആകില്ല. ആദ്യത്തെ തവണ Action റൺ ആയി കഴിഞ്ഞാൽ ഇത് മാനുവൽ ആയി ഓൺ ചെയ്യണം.

* **എവിടെ മാറ്റണം?:** GitHub Settings -> Pages.
* **എന്ത് മാറ്റണം?:** **Branch** എന്നതിൽ **`gh-pages`** എന്ന് സെലക്ട് ചെയ്ത് Save ചെയ്യുക.

---

### (ഓപ്ഷണൽ) ആപ്പിന്റെ ഐഡി മാറ്റുക

ഇത് നിർബന്ധമല്ല, എന്നാലും രണ്ട് ആപ്പുകളും ഒരേ ഫോണിൽ ഇൻസ്റ്റാൾ ചെയ്യാൻ ഇത് മാറ്റുന്നത് നല്ലതാണ്.

* **എവിടെ മാറ്റണം?:** `android/app/build.gradle.kts`
* **എന്ത് മാറ്റണം?:** `applicationId "com.example.sanjid_app"` എന്നത് മാറ്റി പുതിയ പേര് (ഉദാ: `com.example.newproject`) നൽകുക.

### ചുരുക്കത്തിൽ:

നിങ്ങൾ പുതിയൊരു Repository തുടങ്ങിയാൽ:

1. **Settings**-ൽ പോയി "Read and write permissions" കൊടുക്കുക.
2. **`build.yml`**-ൽ വെബ്സൈറ്റ് ലിങ്ക് (Repository Name) തിരുത്തുക.
3. ആദ്യത്തെ ബിൽഡ് കഴിഞ്ഞാൽ **Pages** സെറ്റിംഗ്സിൽ പോയി വെബ്സൈറ്റ് എനേബിൾ ചെയ്യുക.

ഈ മൂന്ന് കാര്യങ്ങൾ ചെയ്താൽ പുതിയ പ്രോജക്റ്റും പഴയത് പോലെ സുഗമമായി പ്രവർത്തിക്കും.

ആകെ **4 സ്ഥലങ്ങളിൽ** മാറ്റം വരുത്തിയാൽ മതി.

---

### 1. ആപ്പിന്റെ പേര് മാറ്റാൻ (App Name)

ഫോണിൽ ഇൻസ്റ്റാൾ ചെയ്യുമ്പോൾ ആപ്പിന് താഴെ കാണുന്ന പേര് മാറ്റാൻ:

* **ഫയൽ:** `android/app/src/main/AndroidManifest.xml`
* **എന്ത് ചെയ്യണം?:**
ഈ ഫയൽ തുറന്ന് `android:label` എന്ന് എഴുതിയ വരി കണ്ടെത്തുക.
```xml
<application
    android:label="പഴയ_പേര്"  <!-- ഇത് മാറ്റി പുതിയ പേര് നൽകുക -->
    android:name="${applicationName}"
    ... >

```


ഇവിടെ "Sanjid Counter" എന്നത് മാറ്റി "My New App" എന്നോ മറ്റോ നൽകുക.

---

### 2. വെബ്സൈറ്റിന്റെ തലക്കെട്ട് മാറ്റാൻ (Web Title)

ബ്രൗസറിൽ തുറക്കുമ്പോൾ ടാബിൽ (Tab) കാണുന്ന പേര് മാറ്റാൻ:

* **ഫയൽ:** `web/index.html`
* **എന്ത് ചെയ്യണം?:**
ഈ ഫയലിൽ `<title>` എന്ന് തുടങ്ങുന്ന വരി കണ്ടെത്തുക.
```html
<title>പഴയ_പേര്</title> <!-- ഇത് മാറ്റി പുതിയ പേര് നൽകുക -->

```



---

### 3. ആപ്പിന്റെ ലോഗോ മാറ്റാൻ (App Icon)

ഏറ്റവും എളുപ്പമുള്ള വഴി താഴെ പറയുന്നതാണ്. കോഡിൽ തൊടേണ്ട ആവശ്യമില്ല.

* **ഫയൽ:** `assets/` ഫോൾഡർ
* **എന്ത് ചെയ്യണം?:**
1. നിങ്ങളുടെ പുതിയ ലോഗോ (ചിത്രം) തയ്യാറാക്കുക.
2. അതിന് **`icon.png`** എന്ന് തന്നെ പേര് നൽകുക (പഴയ ഫയലിന്റെ അതേ പേര്).
3. GitHub-ൽ `assets` ഫോൾഡറിൽ പോയി പഴയ `icon.png` ഡിലീറ്റ് ചെയ്യുക.
4. പുതിയ `icon.png` അപ്‌ലോഡ് ചെയ്യുക.



**എന്തുകൊണ്ട് കോഡ് മാറ്റണ്ട?**
നമ്മുടെ `pubspec.yaml` ഫയലിൽ നേരത്തെ തന്നെ "assets/icon.png എന്ന ചിത്രം എടുത്ത് ഐക്കൺ ആക്കുക" എന്ന് എഴുതി വെച്ചിട്ടുണ്ട്. അതുകൊണ്ട് ചിത്രം മാത്രം മാറ്റിയാൽ മതി, ബാക്കി GitHub Action നോക്കിക്കോളും.

*(നിങ്ങൾ ചിത്രത്തിന്റെ പേര് മാറ്റുകയാണെങ്കിൽ മാത്രം `pubspec.yaml` തുറന്ന് `image_path: "assets/new_name.png"` എന്ന് മാറ്റേണ്ടി വരും).*

---

### 4. സ്പ്ലാഷ് സ്ക്രീൻ മാറ്റാൻ (Splash Screen)

ആപ്പ് തുറക്കുമ്പോൾ കാണുന്ന വെളുത്ത സ്ക്രീനിലെ ചിത്രം മാറ്റാൻ:

* ഇതിനും മുകളിൽ പറഞ്ഞ (3-ാമത്തെ പോയിന്റ്) അതേ വഴി മതി.
* നിങ്ങൾ `assets/icon.png` മാറ്റുന്നതോടെ, സ്പ്ലാഷ് സ്ക്രീനിലെ ലോഗോയും ഓട്ടോമാറ്റിക് ആയി മാറിക്കോളും (കാരണം `pubspec.yaml`-ൽ രണ്ടിനും ഒരേ ചിത്രമാണ് നൽകിയിരിക്കുന്നത്).

---

### ചുരുക്കത്തിൽ (Summary)

പുതിയ പ്രോജക്റ്റ് തുടങ്ങിയാൽ:

1. **`AndroidManifest.xml`** -> ആപ്പിന്റെ പേര് മാറ്റുക.
2. **`web/index.html`** -> വെബ്സൈറ്റ് പേര് മാറ്റുക.
3. **`assets/icon.png`** -> പഴയ ചിത്രം ഡിലീറ്റ് ചെയ്ത് പുതിയത് (അതേ പേരിൽ) അപ്‌ലോഡ് ചെയ്യുക.

ഇത്രയും ചെയ്താൽ പുതിയ ബ്രാൻഡിംഗോട് കൂടി ആപ്പ് റെഡി!
