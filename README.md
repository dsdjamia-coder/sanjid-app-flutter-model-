പുതിയ പ്രോജക്റ്റ് തുടങ്ങുമ്പോൾ ചെയ്യേണ്ട കാര്യങ്ങൾ (Master Guide)

നിലവിലുള്ള Repository കോപ്പി ചെയ്തുകൊണ്ട് (അല്ലെങ്കിൽ Fork ചെയ്ത്) പുതിയൊരു പേരിൽ മറ്റൊരു പ്രോജക്റ്റ് തുടങ്ങുമ്പോൾ, താഴെ പറയുന്ന 5 കാര്യങ്ങൾ മാറ്റിയാൽ മാത്രമേ പുതിയ ആപ്പിന്റെ വെബ്സൈറ്റും APK-യും ശരിയായി ലഭിക്കുകയുള്ളൂ.

1. build.yml ഫയലിലെ മാറ്റം (ഏറ്റവും പ്രധാനം)

നിങ്ങൾ പുതിയ Repository-ക്ക് "new-project" എന്ന് പേര് നൽകി എന്ന് കരുതുക. പക്ഷെ നിങ്ങളുടെ build.yml ഫയലിൽ ഇപ്പോഴും പഴയ ലിങ്ക് ആയിരിക്കും ഉണ്ടാവുക.

എവിടെ മാറ്റണം?: .github/workflows/build.yml

എന്ത് മാറ്റണം?:

വെബ് ലിങ്ക്: run: flutter build web --release --base-href "/sanjid-app-flutter-model-/" എന്ന വരിയിലെ പേര് മാറ്റി പുതിയ Repository-യുടെ പേര് നൽകുക. (ഉദാ: "/new-project/")

APK ഫയലിന്റെ പേര്: run: mv .../app-arm64-v8a-release.apk .../SanjidApp.apk എന്ന വരിയിൽ, SanjidApp.apk എന്നത് മാറ്റി പുതിയ പേര് (ഉദാ: NewApp.apk) നൽകുക.

2. GitHub Settings (പെർമിഷൻ നൽകൽ)

പുതിയ Repository തുടങ്ങുമ്പോൾ പഴയതിന്റെ സെറ്റിംഗ്സ് അതിലേക്ക് കോപ്പി ചെയ്യില്ല. അതിനാൽ "Action" വർക്ക് ആകാൻ പെർമിഷൻ വീണ്ടും നൽകണം.

എവിടെ മാറ്റണം?: GitHub Settings -> Actions -> General -> Workflow permissions.

എന്ത് മാറ്റണം?: "Read and write permissions" എന്നത് വീണ്ടും സെലക്ട് ചെയ്ത് Save ചെയ്യുക. (ഇത് ചെയ്തില്ലെങ്കിൽ APK അപ്‌ലോഡ് ആകില്ല).

3. GitHub Pages (വെബ്സൈറ്റ് ഓൺ ആക്കൽ)

പുതിയ പ്രോജക്റ്റിൽ വെബ്സൈറ്റ് ഓട്ടോമാറ്റിക് ആയി ഓൺ ആകില്ല. ആദ്യത്തെ തവണ Action റൺ ആയി കഴിഞ്ഞാൽ ഇത് മാനുവൽ ആയി ഓൺ ചെയ്യണം.

എവിടെ മാറ്റണം?: GitHub Settings -> Pages.

എന്ത് മാറ്റണം?: Branch എന്നതിൽ gh-pages എന്ന് സെലക്ട് ചെയ്ത് Save ചെയ്യുക.

4. ആപ്പിന്റെ പേരും ഐഡിയും (Branding)

രണ്ട് ആപ്പുകളും ഒരേ ഫോണിൽ ഇൻസ്റ്റാൾ ചെയ്യാൻ ഐഡി മാറ്റണം. ഫോണിൽ കാണുന്ന പേരും മാറ്റണം.

ആപ്പ് ഐഡി: android/app/build.gradle.kts എന്ന ഫയലിൽ applicationId "com.example.sanjid_app" എന്നത് മാറ്റി പുതിയ പേര് (ഉദാ: com.example.newproject) നൽകുക.

ആപ്പിന്റെ പേര്: android/app/src/main/AndroidManifest.xml എന്ന ഫയലിൽ android:label="Old Name" എന്നത് മാറ്റി പുതിയ പേര് നൽകുക.

വെബ് ടൈറ്റിൽ: web/index.html എന്ന ഫയലിൽ <title>Old Name</title> എന്നത് മാറ്റുക.

ഇന്റേണൽ പേര് & വേർഷൻ: pubspec.yaml എന്ന ഫയലിൽ name: old_name എന്നതും version: 1.0.0+1 എന്നതും പുതുക്കുക.

5. ലോഗോ മാറ്റാൻ (Icon)

ഏറ്റവും എളുപ്പമുള്ള വഴി താഴെ പറയുന്നതാണ്. കോഡിൽ തൊടേണ്ട ആവശ്യമില്ല.

ഫയൽ: assets/ ഫോൾഡർ

എന്ത് ചെയ്യണം?:

നിങ്ങളുടെ പുതിയ ലോഗോ (ചിത്രം) തയ്യാറാക്കുക.

അതിന് icon.png എന്ന് തന്നെ പേര് നൽകുക.

GitHub-ൽ assets ഫോൾഡറിൽ പോയി പഴയ icon.png ഡിലീറ്റ് ചെയ്യുക.

പുതിയ icon.png അപ്‌ലോഡ് ചെയ്യുക.

(ശ്രദ്ധിക്കുക: pubspec.yaml-ൽ നമ്മൾ നേരത്തെ സെറ്റ് ചെയ്തത് കൊണ്ട്, ചിത്രം മാറ്റിയാൽ സ്പ്ലാഷ് സ്ക്രീനും ഐക്കണും ഓട്ടോമാറ്റിക് ആയി മാറിക്കോളും).

സംഗ്രഹം (Checklist)

പുതിയൊരു പ്രോജക്റ്റ് തുടങ്ങിയാൽ ഈ ക്രമത്തിൽ ചെയ്യുക:

[ ] Settings -> Permissions കൊടുക്കുക.

[ ] build.yml -> വെബ് ലിങ്കും APK പേരും മാറ്റുക.

[ ] build.gradle.kts -> Application ID മാറ്റുക.

[ ] AndroidManifest.xml & index.html -> പേര് മാറ്റുക.

[ ] assets/icon.png -> ലോഗോ മാറ്റുക.

[ ] ആദ്യത്തെ ബിൽഡ് കഴിഞ്ഞാൽ Pages എനേബിൾ ചെയ്യുക.
